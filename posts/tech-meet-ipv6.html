Transitioning to IPv6: Dual Stack Without Double Trouble

A short reflection on an evening session about moving to IPv6 and how to do it in a safe, realistic way.

The main message of this session was simple: IPv6 is not a “later” topic anymore. A lot of providers and big organisations are already using it. Smaller companies are following too, not because IPv4 suddenly stops working, but because running everything on IPv4 is getting harder and more expensive over time.

I liked that IPv6 was not presented as an easy switch. It changes how networks work. You get a lot more address space, NAT is not the default solution, and end to end connectivity becomes possible again. That can be a good thing, but only if you rebuild your controls properly. IPv6 needs the same attention as IPv4 in routing, firewall rules, monitoring, and documentation.

The most useful part for me was the focus on dual stack. In many real networks, the transition means running IPv4 and IPv6 at the same time. If you do that carelessly, you can create new gaps. One common example is having strict IPv4 firewall rules while IPv6 stays wide open because “we are not really using it yet”. That is exactly how hidden exposure happens.

IPv6 also has its own things you need to understand. Router advertisements, neighbour discovery, and auto configuration are helpful features, but they can also be abused if they are not controlled. A safe baseline is to be explicit about what is allowed, keep a default deny approach where possible, and make sure your logging and monitoring actually covers IPv6 traffic.

After this session, my personal plan is straightforward. First, check what already has IPv6 enabled. Then review DNS readiness, especially AAAA records and resolver settings. After that, make sure firewall policies match on IPv4 and IPv6. Finally, confirm the tools are ready: monitoring, scanning, and incident response should not ignore IPv6. If a company treats IPv6 as “just more addresses”, it can become a blind spot. If they treat it as a planned transition, dual stack becomes a safe bridge.
