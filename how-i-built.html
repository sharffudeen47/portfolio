Why I Kept This Site Simple

A short explanation of how this portfolio is built and why I chose a clean, lightweight setup.

I built this portfolio with plain HTML, CSS, and a small amount of JavaScript. It’s a static site hosted on GitHub Pages.

I chose this approach because I like knowing exactly what is running. When a site has lots of plugins, themes, and extra layers, it can be harder to maintain and easier to break. Keeping it simple makes it faster, clearer, and easier to secure.

Hosting on GitHub Pages also fits the way I like to work. Everything is version controlled, so changes are tracked properly. When I add a new project or write a blog post, it’s a clean update in the repo, and I can always see what changed and when.

Technically, the website uses one main stylesheet for consistent design and a small script for navigation and UI behavior. Each blog post is its own HTML file, so it stays readable and accessible even if JavaScript is turned off.

This setup will scale nicely as I add more content. I can grow the portfolio without dealing with databases or platform changes. It’s simple, practical, and easy to maintain, which is exactly what I want.
