Simplicity as a Security Principle

A look at the technical choices behind this portfolio and why I chose fundamentals over frameworks.

In an era of heavy JavaScript frameworks and bloated CMS platforms, I decided to take a different path for my professional portfolio. This site is built using a "Vanilla" stack: pure HTML, CSS, and JavaScript. It is hosted as a static site on GitHub Pages.

As a cybersecurity student, I believe that knowing exactly what is running on your system is the first step toward securing it. By avoiding third-party templates or drag-and-drop builders, I maintain absolute control over the codebase. This approach eliminates the "hidden" vulnerabilities often found in complex plugins and ensures that the site is incredibly lightweight, fast, and resilient.

I chose GitHub Pages not just for its reliability, but for the workflow discipline it enforces. The entire project is managed via version control. Every blog post reflection and portfolio update is a discrete commit in a repository. This mimics the environment of a professional IT team maintaining a clean folder structure, tracking incremental changes, and ensuring that the "production" environment is always stable and audit-ready.

Technically, the site relies on a shared stylesheet for visual consistency and a minimal JavaScript utility to handle navigation and UI states. Each blog post is its own semantic HTML file, ensuring that the content remains readable and accessible even if scripts are disabled. It’s a design philosophy that mirrors how I approach security: keep it simple, keep it transparent, and keep it maintainable.

This site will continue to grow as I do. Because the structure is modular, I can scale the content indefinitely without ever needing to worry about database overhead or platform migration. It serves as both a repository of my work and a living proof of my commitment to technical fundamentals.
