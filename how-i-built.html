How I Built This Website (Required)

A clear explanation of the technology I used and why I chose it — included to directly meet the e-portfolio evaluation criteria.

I built this e-portfolio as a static website using HTML, CSS, and JavaScript, and I host it on GitHub Pages. I chose this approach because it matches my professional identity as an IT/Cybersecurity student: I want full control over structure, content, and presentation, and I want the site to remain easy to maintain throughout my final year.

GitHub Pages is a strong fit for an e-portfolio because it is reliable, free, and directly connected to a workflow used in real IT teams: version control, clean folder structure, and incremental updates. Each change is tracked in commits, which helps me improve the site over time without breaking working pages. It also makes it simple to add new reflections: each post becomes its own page inside the /posts folder.

I intentionally avoided turning this into a “random personal blog.” Instead, the structure is built around my professional profile and portfolio requirements: About, Portfolio, Blog/Reflections, and Contact. The Blog section contains 300–500 word reflections focused on what I learned and how I will apply it, rather than short captions. This page exists specifically to satisfy the “How I built it” requirement clearly and directly.

Technically, the website uses a shared stylesheet to keep design consistent across all pages and a small JavaScript file to handle navigation behavior (mobile menu + active link highlighting) and simple utilities like the footer year. This keeps the website lightweight, fast, and easy to review. If I expand later, I can add more posts and project pages without changing the overall structure.
